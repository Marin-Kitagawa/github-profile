== Cards Added to This Project

All cards follow the same URL pattern:

[source]
----
https://<your-deployment>/api/<card-name>?param=value&...
----

Common parameters shared by every card:

[cols="1,1,3", options="header"]
|===
| Parameter | Default | Description
| `theme` | `default` | Theme name (60+ built-in themes)
| `dark_theme` | — | Auto-switch to this theme in OS dark mode (`prefers-color-scheme`)
| `title_color` | theme | Card title hex color (without `#`)
| `text_color` | theme | Body text hex color
| `icon_color` | theme | Icon / accent hex color
| `bg_color` | theme | Background color or gradient (`angle,color1,color2,...`)
| `border_color` | theme | Border hex color
| `hide_border` | `false` | Remove card border
| `hide_title` | `false` | Remove card title
| `border_radius` | `4.5` | Corner radius in px
| `disable_animations` | `false` | Turn off SVG animations
| `custom_title` | — | Override the card title text
| `cache_seconds` | varies | CDN cache TTL override
|===

=== GitHub Stats Card (`/api`)

The core card. Shows commits, stars, PRs, issues, and a rank circle.

[source,markdown]
----
![Stats](https://your-deployment/api?username=octocat&theme=github_dark&dark_theme=github_light&show_icons=true&rank_icon=github)
----

Additional parameters:

[cols="1,1,3", options="header"]
|===
| Parameter | Default | Description
| `show_icons` | `false` | Show stat icons
| `hide_rank` | `false` | Hide the rank circle
| `rank_icon` | `default` | `default`, `github`, or `percentile`
| `show` | — | Extra stats: `reviews`, `prs_merged`, `prs_merged_percentage`, `discussions_started`, `discussions_answered`
| `hide` | — | Hide specific stats: `stars`, `commits`, `prs`, `issues`, `contribs`
| `include_all_commits` | `false` | Count commits across all years (uses REST API)
| `dark_theme` | — | *New:* theme name to use when OS is in dark mode
|===

=== Auto Dark Mode

Any card can now automatically switch theme when the viewer's OS is in dark mode:

[source,markdown]
----
![Stats](https://your-deployment/api?username=octocat&theme=github_light&dark_theme=github_dark)
----

This works because the generated SVG embeds a `@media (prefers-color-scheme: dark)` CSS block. GitHub renders profile images as `<img>` elements, which _do_ respect CSS media queries inside embedded SVGs in modern browsers.

=== Contribution Animations

==== Contribution Snake

A snake animation that eats your contribution squares. Runs via GitHub Actions and commits an SVG to your repo.

Setup — add `.github/workflows/snake.yml` to your _profile_ repository:

[source,yaml]
----
name: Generate Contribution Snake
on:
  schedule:
    - cron: "0 0 * * *"
  workflow_dispatch:
jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: Platane/snk@v3
        with:
          github_user_token: ${{ secrets.GITHUB_TOKEN }}
          outputs: |
            dist/snake.svg
            dist/snake-dark.svg?palette=github-dark
      - uses: crazy-max/ghaction-github-pages@v4
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
----

Embed with automatic light/dark switching:

[source,markdown]
----
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/USERNAME/USERNAME/output/snake-dark.svg" />
  <source media="(prefers-color-scheme: light)" srcset="https://raw.githubusercontent.com/USERNAME/USERNAME/output/snake.svg" />
  <img src="https://raw.githubusercontent.com/USERNAME/USERNAME/output/snake.svg" />
</picture>
----

Source: https://github.com/Platane/snk

=== GitHub Actions — Auto-updating README Sections

These actions commit updated content directly to your README on a schedule. Wrap target sections with HTML comment markers they look for.

==== GitHub Activity Feed

Appends your 10 most recent GitHub events (pushes, PRs, reviews, comments) to your README.

[source,markdown]
----
<!--START_SECTION:activity-->
<!--END_SECTION:activity-->
----

[source,yaml]
----
- uses: jamesgeorge007/github-activity-readme@master
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  with:
    MAX_LINES: 10
----

Source: https://github.com/jamesgeorge007/github-activity-readme

==== WakaTime Coding Stats (Text)

Embeds a formatted WakaTime breakdown (languages, editors, OS, time) directly into your README as text — complementary to the WakaTime SVG card.

[source,markdown]
----

<!--END_SECTION:waka-->
----

=== Comprehensive Metrics (lowlighter/metrics)

The most powerful single tool in this space. Generates a complete metrics SVG covering 30+ data sources via a single GitHub Action, including:

* GitHub stats, languages, streak
* Pagespeed / website performance
* Habits (coding time, peak hours, commit patterns)
* Achievements and milestones
* Followers network graph
* Topics and interests
* RSS / blog posts
* Music (Spotify, Last.fm)
* LeetCode, Codeforces, Chess.com, and more

[source,yaml]
----
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    user: YOUR_USERNAME
    template: classic
    base: header, activity, community, repositories, metadata
    plugin_habits: yes
    plugin_achievements: yes
    plugin_languages: yes
    plugin_wakatime: yes
    plugin_wakatime_token: ${{ secrets.WAKATIME_API_KEY }}
----

NOTE: This project and `lowlighter/metrics` are complementary. This project generates on-demand URL-embeddable SVGs; `metrics` generates richer static outputs via Actions.

Source: https://github.com/lowlighter/metrics

== API Deployment Notes

=== Environment Variables

[cols="1,1,3", options="header"]
|===
| Variable | Required | Purpose
| `PAT_1` | Yes | GitHub Personal Access Token (read:user, repo scopes)
| `WAKATIME_API_KEY` | For WakaTime | Used in the GitHub Action only; not needed server-side
| `CACHE_SECONDS` | No | Override all card TTLs globally
| `FETCH_MULTI_PAGE_STARS` | No | Set `true` to paginate beyond 100 repos for star counts
|===

=== Cache TTLs

[cols="1,1,1,1", options="header"]
|===
| Card | Default | Min | Max
| Stats, Streak, Heatmap, LeetCode, Stack Overflow | 1 day | 12 hours | 2 days
| Top Languages, NPM, PyPI | 6 days | 2 days | 10 days
| Repo Pin, Gist | 10 days | 1 day | 10 days
| WakaTime, Last.fm | 1 day | 12 hours | 2 days
| Any error response | 10 minutes | — | —
|===

=== Rate Limits

[cols="1,2", options="header"]
|===
| Source | Notes
| GitHub GraphQL | 5,000 points/hour per token. Use multiple `PAT_N` tokens for high traffic.
| LeetCode GraphQL | Unauthenticated public API — generous limits for profile data.
| Stack Overflow REST | 300 requests/day without a key; 10,000/day with a registered app key.
| NPM Registry | No authentication required. No documented hard limit.
| pypistats.org | No authentication required. 15 req/min unofficial soft limit.
| Last.fm API | 5 req/sec; free API key sufficient for all profile card usage.
|===
