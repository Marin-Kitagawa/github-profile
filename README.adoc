A self-hosted Vercel service that generates dynamic SVG stat cards for GitHub profile READMEs.
Embed live GitHub data, coding activity, competitive programming stats, package metrics, and more — all as theme-aware SVG images.

== Quick Start

All cards are served from a single Vercel deployment as `image/svg+xml` responses.
Embed them in any Markdown or AsciiDoc file:

[source,markdown]
----
![GitHub Stats](https://YOUR_DEPLOYMENT/api?username=octocat&theme=github_dark)
----

[source,asciidoc]
----
image::https://YOUR_DEPLOYMENT/api?username=octocat&theme=github_dark[GitHub Stats]
----

== Deploy Your Own Instance

=== Prerequisites

* A https://vercel.com[Vercel] account (free tier is sufficient)
* A GitHub https://github.com/settings/tokens[Personal Access Token] with scopes:
** `read:user`
** `repo` (needed for private repo star counts; omit if not required)

=== Steps

1. Fork this repository to your GitHub account.
2. Import the fork into Vercel from the dashboard.
3. Add the environment variable:
+
[cols="1,3", options="header"]
|===
| Variable | Value
| `PAT_1` | Your GitHub Personal Access Token
|===
4. Deploy. Your instance URL becomes the base for all card URLs.

TIP: For high-traffic instances, add `PAT_2` through `PAT_9` to round-robin across multiple tokens and avoid rate limits.

=== Optional Environment Variables

[cols="1,1,3", options="header"]
|===
| Variable | Default | Description
| `CACHE_SECONDS` | per-card | Override TTL for every card globally
| `FETCH_MULTI_PAGE_STARS` | `false` | Set `true` to paginate beyond the first 100 repos when counting stars
|===

== Cards

=== Common Parameters

Every card accepts these query parameters:

[cols="1,1,3", options="header"]
|===
| Parameter | Default | Description
| `theme` | `default` | Built-in theme name (see <<themes>>)
| `dark_theme` | — | Theme to use when the viewer's OS is in dark mode (`prefers-color-scheme`)
| `title_color` | theme | Card title hex color (omit the `#`)
| `text_color` | theme | Body text hex color
| `icon_color` | theme | Icon and accent hex color
| `bg_color` | theme | Background color or gradient (see <<gradients>>)
| `border_color` | theme | Border hex color
| `ring_color` | theme | Rank-ring accent color (stats card only)
| `hide_border` | `false` | Remove the card border
| `hide_title` | `false` | Remove the card title bar
| `border_radius` | `4.5` | Corner radius in pixels
| `disable_animations` | `false` | Disable all SVG animations
| `custom_title` | — | Override the title text
| `cache_seconds` | varies | Override the CDN cache TTL for this card
|===

[[gradients]]
==== Gradient Backgrounds

Pass `bg_color` as `angle,color1,color2[,color3...]` (hex colors without `#`):

[source]
----
bg_color=45,0d1117,1a1b27,161b22

=== GitHub Stats Card

Endpoint: `GET /api`

[source,markdown]
----
![Stats](https://YOUR_DEPLOYMENT/api?username=octocat&show_icons=true&theme=github_dark&rank_icon=github)
----

image::https://github-profile-demo.vercel.app/api?username=octocat&show_icons=true&theme=github_dark&rank_icon=github[Stats Card Preview]

==== Parameters

[cols="1,1,3", options="header"]
|===
| Parameter | Default | Description
| `username` | — | *Required.* GitHub username
| `show_icons` | `false` | Show stat icons
| `hide_rank` | `false` | Hide the rank circle
| `rank_icon` | `default` | Icon style: `default` (letter grade), `github` (Octocat), `percentile`
| `show` | — | Comma-separated extra stats: `reviews`, `prs_merged`, `prs_merged_percentage`, `discussions_started`, `discussions_answered`
| `hide` | — | Comma-separated stats to hide: `stars`, `commits`, `prs`, `issues`, `contribs`
| `include_all_commits` | `false` | Count commits across all years via REST API (slower)
| `commits_year` | — | Restrict commit count to a specific year (e.g. `2024`)
| `line_height` | `25` | Vertical spacing between stat rows (px)
| `card_width` | `450` | Card width in pixels
| `text_bold` | `true` | Bold stat labels
| `number_format` | `short` | `short` (1k) or `long` (1000)
| `number_precision` | — | Decimal places for formatted numbers (0–2)
| `locale` | `en` | UI language (30+ locales supported)
| `dark_theme` | — | Auto dark-mode theme name
|===

==== Rank System

Ranks are calculated from a weighted percentile across commits, PRs, reviews, issues, stars, and followers.

[cols="1,1", options="header"]
|===
| Level | Percentile
| S | Top 1%
| A+ | Top 12.5%
| A | Top 25%
| A- | Top 37.5%
| B+ | Top 50%
| B | Top 62.5%
| B- | Top 75%
| C+ | Top 87.5%
| C | Top 100%
|===

== Dark Mode Auto-Detection

Any card can automatically switch theme when the viewer's OS is in dark mode, using the `dark_theme` parameter.
The generated SVG embeds a `@media (prefers-color-scheme: dark)` CSS block — this works because GitHub renders profile images as `<img>` elements, which honour CSS media queries inside SVGs.

[source,markdown]
----
![Stats](https://YOUR_DEPLOYMENT/api?username=octocat&theme=github_light&dark_theme=github_dark&show_icons=true)
----

Combine any light theme with any dark theme:

[cols="1,1", options="header"]
|===
| Light (`theme=`) | Recommended dark pair (`dark_theme=`)
| `github_light` | `github_dark`
| `default` | `dark`
| `solarized-light` | `solarized-dark`
| `gruvbox_light` | `gruvbox`
| `catppuccin_latte` | `catppuccin_mocha`
|===

== GitHub Actions

Two workflows are bundled for your *profile repository* (the repo named `YOUR_USERNAME/YOUR_USERNAME`).

=== Contribution Snake

`.github/workflows/snake.yml` — runs daily and pushes an SVG snake animation to the `output` branch.

[source,yaml]
----
on:
  schedule:
    - cron: "0 0 * * *"
  workflow_dispatch:
----

Embed with automatic light/dark switching:

[source,markdown]
----
<picture>
  <source media="(prefers-color-scheme: dark)"
          srcset="https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_USERNAME/output/github-contribution-grid-snake-dark.svg" />
  <source media="(prefers-color-scheme: light)"
          srcset="https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_USERNAME/output/github-contribution-grid-snake.svg" />
  <img src="https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_USERNAME/output/github-contribution-grid-snake.svg" />
</picture>
----

=== Auto-Update README

`.github/workflows/update-readme.yml` — updates activity feed and WakaTime stats sections every 6 hours.

Add these marker comments to your profile README:

[source,markdown]
----
<!--START_SECTION:activity-->
<!--END_SECTION:activity-->

<!--START_SECTION:waka-->
<!--END_SECTION:waka-->
----

Requires the `WAKATIME_API_KEY` secret to be set for the WakaTime section.

== Access Control

Self-hosted instances can restrict which users may be queried.

[cols="1,3", options="header"]
|===
| Variable | Effect
| `ALLOWED_USERS` | Comma-separated whitelist of GitHub usernames. Requests for other users return an error card.
| `BLOCKED_USERS` | Comma-separated blacklist. Listed users always receive an error card.
|===

== Cache TTLs

[cols="1,1,1,1", options="header"]
|===
| Card | Default | Min | Max
| Stats, Streak, Heatmap | 1 day | 12 hours | 2 days
| LeetCode, Stack Overflow, Last.fm, WakaTime | 1 day | 12 hours | 2 days
| Top Languages, NPM, PyPI | 6 days | 2 days | 10 days
| Repo Pin, Gist | 10 days | 1 day | 10 days
| Any error response | 10 minutes | — | —
|===

Override per-request with `?cache_seconds=N` (clamped to the card's min/max).

== Rate Limits

[cols="1,2", options="header"]
|===
| Source | Notes
| GitHub GraphQL | 5,000 points/hour per PAT. Add `PAT_2`–`PAT_9` for round-robin.
| LeetCode GraphQL | Public unauthenticated API; generous limits for profile data.
| Stack Overflow REST | 300 req/day without a key; 10,000/day with a registered app key.
| NPM Registry | No auth required; no documented hard limit.
| pypistats.org | No auth required; ~15 req/min soft limit.
| Last.fm API | 5 req/sec; free API key sufficient for all profile card usage.
|===

=== Running Locally

[source,bash]
----
git clone https://github.com/YOUR_USERNAME/github-profile.git
cd github-profile
npm install
cp .env.example .env      # add your PAT_1
node express.js           # serves on http://localhost:9000
----

Test a card:

[source]
----
http://localhost:9000/api?username=octocat&show_icons=true&theme=dark
----
